# Research Methodologies Reference

## BMAD-Inspired Research Framework

### Three-Tier Research Approach

#### Tier 1: Foundational Research (80% effort)

**Purpose:** Establish baseline domain understanding
**Methods:**

- Market sizing and segmentation analysis
- Competitive landscape mapping
- Regulatory environment assessment
- Technology stack evaluation
- Customer journey investigation

**Time Investment:** 15-20 hours
**Output Quality:** Comprehensive but not differentiated

#### Tier 2: Differentiated Intelligence (15% effort)

**Purpose:** Uncover non-obvious insights
**Methods:**

- Primary source interviews with domain experts
- Patent landscape analysis for emerging technologies
- Financial model reverse engineering
- Customer switching cost analysis
- Partnership and M&A pattern analysis

**Time Investment:** 3-5 hours
**Output Quality:** Competitive advantage potential

#### Tier 3: Proprietary Insights (5% effort)

**Purpose:** Generate unique strategic perspectives
**Methods:**

- Cross-domain pattern application
- Emerging technology trajectory modeling
- Regulatory change impact simulation
- Customer behavior prediction modeling
- White space opportunity identification

**Time Investment:** 1-2 hours
**Output Quality:** Strategic differentiation

### Expert Interview Methodologies

#### ZOPA Interview Framework

**Zone of Possible Agreement Identification:**

1. **Zone Discovery** (15 min)
   - Expert's area of specialization
   - Current focus and recent changes
   - Confidence boundaries identification

2. **Opinion Formation** (20 min)
   - Structured opinion elicitation
   - Confidence interval assignment
   - Alternative perspective exploration

3. **Prediction Assessment** (15 min)
   - Future scenario evaluation
   - Probability assignment
   - Timeline estimation

4. **Assumption Testing** (10 min)
   - Hypothesis validation/refutation
   - Blind spot identification
   - Contrarian viewpoint exploration

#### Cognitive Bias Mitigation

**Interview Design Principles:**

- **Anchoring Bias:** Ask for ranges before point estimates
- **Confirmation Bias:** Present contrary evidence for reaction
- **Availability Heuristic:** Ask for base rate information
- **Overconfidence Bias:** Request confidence intervals
- **Recency Bias:** Balance recent vs. historical examples

## Research Quality Assessment

### Source Credibility Matrix

| Source Type             | Credibility Factors                          | Quality Indicators                         |
| ----------------------- | -------------------------------------------- | ------------------------------------------ |
| **Academic Research**   | Peer review, citation count, journal impact  | Methodology rigor, replication studies     |
| **Industry Reports**    | Analyst reputation, methodology transparency | Sample size, data recency, bias disclosure |
| **Expert Interviews**   | Domain authority, position, independence     | Specific knowledge, balanced perspective   |
| **Company Disclosures** | Regulatory oversight, audit requirements     | Consistency, materiality, forward-looking  |
| **Patent Filings**      | Legal scrutiny, technical depth              | Filing quality, prosecution history        |

### Information Triangulation Protocol

#### Three-Source Validation

**Requirement:** Every key finding validated by minimum 3 independent sources
**Source Types:** Must include at least 2 different source categories
**Conflict Resolution:** Investigate discrepancies through additional primary research

#### Confidence Scoring

```
High Confidence (8-10):
├── 3+ independent source confirmation
├── Recent information (≤6 months)
├── Primary source validation
└── Expert consensus

Medium Confidence (5-7):
├── 2 independent source confirmation
├── Moderately recent (6-18 months)
├── Mixed primary/secondary sources
└── Expert uncertainty

Low Confidence (1-4):
├── Single source or speculation
├── Outdated information (>18 months)
├── Secondary sources only
└── Expert disagreement
```

#### Research Database Structure

**Recommended Schema:**

```sql
-- Companies table
CREATE TABLE companies (
    id UUID PRIMARY KEY,
    name VARCHAR NOT NULL,
    domain VARCHAR,
    founded_date DATE,
    funding_total DECIMAL,
    employee_count INTEGER,
    headquarters VARCHAR,
    updated_at TIMESTAMP
);

-- Research_findings table
CREATE TABLE research_findings (
    id UUID PRIMARY KEY,
    company_id UUID REFERENCES companies(id),
    finding_type VARCHAR, -- 'technology', 'market', 'financial', etc.
    description TEXT,
    source_url VARCHAR,
    confidence_score INTEGER, -- 1-10
    created_at TIMESTAMP
);

-- Expert_interviews table
CREATE TABLE expert_interviews (
    id UUID PRIMARY KEY,
    expert_name VARCHAR,
    expert_title VARCHAR,
    interview_date DATE,
    key_insights TEXT[],
    confidence_scores JSONB
);
```

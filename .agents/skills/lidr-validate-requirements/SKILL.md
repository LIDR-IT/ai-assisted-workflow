# Requirements Validator & Traceability Matrix Generator

Phase: 3 — Specification | Gate: 2 (final validation) | Language: Spanish

## Workflow

### Automated 5-Pass Validation (NEW)

1. **Execute PRD Parser**: `scripts/prd-parser.py` extracts functionalities and NFR categories from PRDs
2. **Execute RTM Generator**: `scripts/rtm-generator.py` performs automated 5-pass validation
3. **Review Results**: Human validation of automated findings and gap prioritization
4. **Generate Reports**: RTM, Gap Report, and Implementation Clusters automatically generated
5. **Gate 2 Decision**: Based on automated criteria plus human judgment

### Legacy Manual Process (Fallback)

1. Load all RFs generated by `generate-rf` (manual)
2. Load all NFRs generated by `generate-nfr` (manual)
3. Load PRD-F §2.4 (Funcionalidades Clave) — the source of functional scope (manual)
4. Load PRD-T §3 (Arquitectura Propuesta) — the source of technical scope (manual)
5. Execute 5 validation passes (manual - see legacy section)
6. Generate Requirements Traceability Matrix (RTM) using `templates/rtm.md` (manual)
7. Generate Implementation Clusters for Sprint Planning (manual)
8. Generate Gap Report with action items (manual)

### Decision Point

- If automated scripts succeed → Continue with automated results + human review
- If automated scripts fail → Fallback to manual process
- Either path leads to: If PASS → Ready for `epic-breakdown` and Gate 2

## Automation Scripts

### `scripts/prd-parser.py` — PRD Analysis and Requirements Extraction

**Purpose**: Systematically parses PRD documents to extract functionalities, NFR categories, and validation context.

**Key Features**:

- Auto-discovers PRD files in multiple locations (`docs/`, `docs/projects/`, skill outputs)
- Extracts functionalities from §2.4 with IDs, titles, priorities, acceptance criteria
- Identifies NFR categories from §5 and architecture decisions
- Validates mandatory NFR categories for software projects
- Generates JSON output for automated processing + human-readable summary

**Usage**:

```bash
# Auto-discovery mode (recommended)
python scripts/prd-parser.py --output-dir validation-results

# Explicit paths
python scripts/prd-parser.py \
  --prd-funcional docs/projects/PRD-Funcional.md \
  --prd-tecnico docs/projects/PRD-Tecnico.md \
  --output-dir validation-results
```

**Outputs**:

- `prd-analysis.json`: Structured data for automated validation
- `prd-summary.md`: Human-readable summary with statistics

### `scripts/rtm-generator.py` — 5-Pass Automated Validation

**Purpose**: Executes comprehensive 5-pass validation and generates RTM + Gap Reports automatically.

**Key Features**:

- **Pass 1**: Functional Coverage (PRD-F → RFs) — automated gap detection
- **Pass 2**: Technical Coverage (PRD-T → NFRs) — mandatory category validation
- **Pass 3**: NFR Allocation (NFRs ↔ RFs) — orphan detection and mapping validation
- **Pass 4**: Internal Coherence — contradiction detection and circular dependency analysis
- **Pass 5**: Testability & Sprint Readiness — BDD scenario validation and measurability checks
- Generates Gate 2 readiness assessment with PASS/FAIL/CONDITIONAL status

**Usage**:

```bash
# Full automated validation
python scripts/rtm-generator.py \
  --project-dir . \
  --output-dir validation-results \
  --verbose

# Generate reports only
python scripts/rtm-generator.py \
  --rtm-file rtm-report.md \
  --gap-file gap-analysis.md
```

**Outputs**:

- `rtm.md`: Complete Requirements Traceability Matrix
- `gap-report.md`: Detailed gap analysis with action items and implementation clusters
- `validation-results.json`: Structured results for further processing and metrics

### Pass Details (Automated Logic)

### Pass 1: Functional Coverage (PRD-F → RFs) ✅ AUTOMATED

**Algorithm**:

```python
For each Functionality in PRD-F §2.4:
  covering_rfs = [rf for rf in all_rfs if functionality.id in rf.source]
  if len(covering_rfs) == 0:
    → GAP: "Funcionalidad {F-XXX} sin RF asociado"
  elif len(covering_rfs) == 1:
    → WARN: "Funcionalidad {F-XXX} posible sub-descomposición"
```

**Automated Detection**:

- Parses PRD-F §2.4 for F-XXX-YYY patterns
- Cross-references with RF source fields
- Generates specific gap reports with owner assignment

### Pass 2: Technical Coverage (PRD-T → NFRs) ✅ AUTOMATED

**Algorithm**:

```python
mandatory_categories = ['security', 'performance', 'availability', 'privacy', 'compliance']
For each category in PRD-T §5 + mandatory_categories:
  category_nfrs = [nfr for nfr in all_nfrs if category in nfr.category]
  if len(category_nfrs) == 0:
    if category in mandatory_categories:
      → CRITICAL: "Categoría obligatoria {category} faltante"
    else:
      → GAP: "Categoría {category} mencionada sin NFR"
```

**Automated Detection**:

- Keyword-based category detection in PRD-T
- Domain-specific mandatory category enforcement
- Architecture decision implicit NFR detection

### Pass 3: NFR Allocation (NFRs ↔ RFs) ✅ AUTOMATED

**Algorithm**:

```python
For each NFR:
  if not nfr.system_wide and len(nfr.applicable_rfs) == 0:
    → GAP: "NFR {nfr.id} huérfano - sin mapeo a RFs"

For each RF:
  applicable_nfrs = [nfr for nfr where rf.id in nfr.applicable_rfs or nfr.system_wide]
  if len(applicable_nfrs) == 0:
    → WARN: "RF {rf.id} sin NFRs aplicables"
```

**Automated Detection**:

- Parses NFR applicable_rfs fields and system_wide flags
- Cross-references RF-NFR mappings
- Identifies orphan NFRs and under-constrained RFs

### Pass 4: Internal Coherence (RFs ↔ RFs, NFRs ↔ NFRs) ✅ AUTOMATED

**Algorithm**:

```python
# Circular dependency detection
def has_circular_dependency(rf_id, visited, path):
  if rf_id in path: return True  # Circular found
  for dep in rf.dependencies:
    if has_circular_dependency(dep, visited, path): return True

# Contradiction detection (keyword-based)
contradiction_keywords = [('allow', 'deny'), ('enable', 'disable'), ...]
for rf1, rf2 in all_rf_pairs:
  if detect_contradiction(rf1.description, rf2.description):
    → CRITICAL: "Contradicción entre {rf1.id} vs {rf2.id}"
```

**Automated Detection**:

- Graph-based circular dependency analysis
- Keyword-based contradiction detection
- Cross-validation against rf-coherence.md checklist

### Pass 5: Testability & Sprint Readiness ✅ AUTOMATED

**Algorithm**:

```python
For each RF:
  bdd_scenarios = extract_bdd_scenarios(rf.content)
  if len(bdd_scenarios) < 3:
    → GAP: "RF {rf.id} necesita ≥3 escenarios BDD"

For each NFR:
  if not (nfr.metric and nfr.threshold and nfr.validation_method):
    → GAP: "NFR {nfr.id} no medible - falta métrica/umbral/validación"

# Implementation clustering
clusters = group_rfs_by_dependencies_and_category(all_rfs)
for cluster in clusters:
  estimate_sprint_effort(cluster)
```

**Automated Detection**:

- BDD scenario extraction and counting (Given/When/Then patterns)
- NFR measurability validation (metric + threshold + method required)
- Automatic clustering by dependencies and categories
- Sprint effort estimation based on RF complexity

- ❌ **CRITICAL BLOCKER**: Missing security NFRs in any software project

- ⚠️ **WARNING**: <80% BDD coverage (can proceed with plan to complete in Sprint 1)
- ⚠️ **WARNING**: Under-decomposed functionalities (single RF per functionality)

## Enhanced Key Rules (Updated for Automation)

### Automation-First Approach

- **Always attempt automated validation first** — only fallback to manual if scripts fail

### Quality Gates Integration

- **Automated validation results** are input to `/advance-gate 2` command
- **PASS/CONDITIONAL status required** before epic breakdown phase
